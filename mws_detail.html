03 Design System

MWS
设计规范体系

美团云统一设计语言与组件资产库，根据美团 MTD 原子级的规范搭建 MWS 业务组件规范，解决 UI 风格分散、研发还原度低、跨团队协同成本极高的核心问题。

30+规范文档页面
100+Icon 资产
40+覆盖系统/产品线
Background
背景 — 为什么要做 MWS？
🧩

核心痛点

美团云旗下 IaaS、SaaS、SRE 等多条产品线各自为政，UI 风格高度不统一。设计资产分散、无法复用，研发还原度低，每个项目都要"从头来一遍"。跨团队协同成本极高，设计师与研发的沟通摩擦严重影响交付效率。

🎯

产品目标

统一语言：为美团云所有 B 端产品建立一套共同的视觉设计语言与交互规范。

提效交付：通过组件资产库与交付标准，减少设计→研发的重复沟通。

场景沉淀：从通用组件规范延伸到业务场景，形成可复用的最佳实践资产。

生态建设：推动设计文化在云产品团队中的落地与传承。

System Architecture
三层体系：从底层到场景
设计基础
→
组件规范
→
业务模板
→
场景沉淀
→
研发交付
🎨

第一层：设计基础

色彩系统、字体规范、间距网格、图标体系（400+ Icon）、阴影与圆角。建立视觉语言的"最小公约数"，确保全局一致性。

🔧

第二层：组件规范

表单、表格、导航、弹窗、提示、数据可视化等核心 UI 组件的交互规范与视觉标准，覆盖各状态与异常场景。

📐

第三层：场景沉淀

将 IaaS 资源管理、监控大盘、配置中心等高频业务场景提炼为可复用的页面模板，降低新项目启动成本。

Core Work
四大核心工作模块
为美团云建立第一套统一视觉语言，解决多产品线"各说各话"的混乱局面，奠定整个规范体系的基础。
🎨

色彩系统

建立品牌色、功能色（成功/警告/错误/信息）、中性色的完整语义色板。统一深色/浅色背景下的对比度标准，确保无障碍可访问性（WCAG AA 标准）。

🔡

字体与排版

制定标题（H1-H4）、正文、辅助信息、标签的字号/字重/行高规范。在信息密度极高的 B 端场景下，确保层级清晰、阅读舒适。

图标体系建设
📦

400+ 图标资产

从零构建云产品专属图标库，覆盖服务器、网络、存储、安全、监控等核心业务场景，统一线条风格与视觉重量。

📏

栅格与间距

定义 8px 基准网格系统，规范组件内外间距、页面布局间距，从根本上消除设计稿与研发实现的像素偏差。

🌙

暗色模式支持

完整的 Token 体系支持明暗双主题切换，让色彩系统具备良好的可扩展性，为后续主题定制预留接口。

输出 30+ 页组件规范文档，覆盖从基础元素到复杂交互模式的完整规范，成为设计与研发之间的唯一"语言字典"。
核心组件规范覆盖
数据展示
数据表格

规范水平型与网格型两种表格样式，定义对齐方式、数据展示格式、颜色筛选、编辑态、hover 信息与适配规则，覆盖 B 端最高频数据展示场景。

页面结构
标题区块

统一页面标题区的组合规则，覆盖简单标题、详情页标题、表单区三种形态，规范操作按钮位置、面包屑展示与页面层级关系。

页面结构
Tab 区块

定义水平一级、二级、三级 Tab 的嵌套使用规范，区分内容 Tab 与功能 Tab，规范各层级配套表格的列宽、操作列与分页交互。

导航结构
MWS 导航

覆盖全局导航、多 Tab/AppKey 切换、环境切换、告警入口、帮助中心等顶导模式，适配美团云一级/二级/三级/四级多层级导航结构与侧边栏展开收起。

数据录入
用户信息

规范头像尺寸代码、单行与多行信息展示、交互卡片弹出样式，统一 B 端人员信息的视觉呈现与 hover 交互行为。

流程引导
步骤条

定义进行中、已完成、错误三种状态的视觉规范，区分流程型与状态型两种形态，规范步骤条与配套操作区的联动展示方式。

数据分析
对比组件

用于配置项变更前后的差异对比场景，定义常用样式与应用示例，规范高亮标注、新增/删除行的视觉区分方式。

数据录入
穿梭框

支持基础样式、用户信息、表格等多种数据类型的穿梭选择，规范单选/多选状态、搜索筛选、已选列表的交互形态，覆盖应用示例场景。

基于 MWS 规范沉淀 4 类标准模板页面，覆盖 B 端最高频场景，研发直接套用模板即可完成页面搭建，无需从零设计。
模板页面类型
列表页
列表页模板

覆盖基础列表、卡片列表、活动列表、卡片列表四种形态，规范筛选区、操作区、分页区的标准布局，以及行内操作、批量操作的交互模式。

表单页
表单页模板

提供基础表单（单栏/双栏）、分组表单、分步表单、抽屉表单五种形态，规范必填标注、校验时机、操作按钮位置，适配新建、编辑、配置等场景。

详情页
详情页模板

覆盖基础详情、分组详情、带 Tab 文本详情、带 Tab 列表详情、组合信息详情五种形态，统一字段展示密度、分区规则与操作区位置。

Dashboard
Dashboard 页模板

提供监控型与分析型两种大盘形态，规范指标卡片、趋势图、明细表的组合布局，以及时间筛选、维度切换、数据下钻的交互规范。

建立设计师与研发的协同工作流，通过统一标注规范和组件文档，大幅降低研发还原成本，实现设计交付标准化。
📐

标注规范统一

制定间距、字号、颜色的标注规则，设计稿输出格式标准化，研发按规范还原，减少反复沟通。

📚

组件文档体系

每个组件配套使用说明、交互说明、状态说明，形成可检索的设计知识库，新成员上手成本降低 60%。

交付流程优化
Before

改版前 — 高沟通成本

设计师口头描述交互逻辑，研发自行理解，还原偏差大，需多轮 Review 返工，平均一个页面需要 3-5 次反复确认。

After

改版后 — 规范驱动交付

设计稿标注完整，组件文档自助查阅，研发直接对照组件库还原，Review 轮次从 3-5 次降至 1-2 次，交付效率提升 40%+。

设计体系架构师

从 0 开始规划 MWS 设计规范的整体框架，定义设计原则、颜色体系、字体规范、间距系统，制定组件分类标准，搭建完整设计语言基础。

主导 30+ 页组件规范文档与 400+ 图标资产的制作，与研发团队深度协作，推动设计资产直接进入前端组件库，实现设计与研发的一致性。

🚀

推广与落地协同

主动推动设计规范在多个业务线的落地推广，培训设计师使用规范，组织设计走查，持续迭代规范内容，形成设计规范的自我完善机制。

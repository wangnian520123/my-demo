03 Design System B端体验 0→1

MWS
设计规范体系

美团云统一设计语言与组件资产库，从 0 到 1 搭建覆盖多条产品线的 B 端设计规范，解决 UI 风格分散、研发还原度低、跨团队协同成本极高的核心问题。

100+规范文档页面
400+Icon 资产
20+覆盖系统/产品线
20%+信息密度提升
Background
背景 — 为什么要做 MWS？
🧩

核心痛点

美团云旗下 IaaS、SaaS、SRE 等多条产品线各自为政，UI 风格高度不统一。设计资产分散、无法复用，研发还原度低，每个项目都要"从头来一遍"。跨团队协同成本极高，设计师与研发的沟通摩擦严重影响交付效率。

🎯

产品目标

统一语言：为美团云所有 B 端产品建立一套共同的视觉设计语言与交互规范。

提效交付：通过组件资产库与交付标准，减少设计→研发的重复沟通。

场景沉淀：从通用组件规范延伸到业务场景，形成可复用的最佳实践资产。

生态建设：推动设计文化在云产品团队中的落地与传承。

System Architecture
三层体系：从底层到场景
设计基础
→
组件规范
→
业务模板
→
场景沉淀

研发交付
🎨

第一层：设计基础

色彩系统、字体规范、间距网格、图标体系（400+ Icon）、阴影与圆角。建立视觉语言的"最小公约数"，确保全局一致性。

🔧

第二层：组件规范

表单、表格、导航、弹窗、提示、数据可视化等核心 UI 组件的交互规范与视觉标准，覆盖各状态与异常场景。

📐

第三层：场景沉淀

将 IaaS 资源管理、监控大盘、配置中心等高频业务场景提炼为可复用的页面模板，降低新项目启动成本。

Core Work
四大核心工作模块
为美团云建立第一套统一视觉语言，解决多产品线"各说各话"的混乱局面，奠定整个规范体系的基础。
🎨

色彩系统

建立品牌色、功能色（成功/警告/错误/信息）、中性色的完整语义色板。统一深色/浅色背景下的对比度标准，确保无障碍可访问性（WCAG AA 标准）。

🔡

字体与排版

制定标题（H1-H4）、正文、辅助信息、标签的字号/字重/行高规范。在信息密度极高的 B 端场景下，确保层级清晰、阅读舒适。

图标体系建设
📦

400+ 图标资产

从零构建云产品专属图标库，覆盖服务器、网络、存储、安全、监控等核心业务场景，统一线条风格与视觉重量。

📏

栅格与间距

定义 8px 基准网格系统，规范组件内外间距、页面布局间距，从根本上消除设计稿与研发实现的像素偏差。

🌙

暗色模式支持

完整的 Token 体系支持明暗双主题切换，让色彩系统具备良好的可扩展性，为后续主题定制预留接口。

输出 100+ 页组件规范文档，覆盖从基础元素到复杂交互模式的完整规范，成为设计与研发之间的唯一"语言字典"。
核心组件规范覆盖
数据录入
表单 · 输入框 · 选择器

规范各类输入态（默认/聚焦/错误/禁用/只读）、校验规则展示、必填标注方式，以及复杂表单的分组与折叠模式。

数据展示
表格 · 列表 · 详情卡片

B 端最高频的数据展示场景。定义列宽分配策略、操作列规范、空状态设计、分页交互，以及超长内容的截断与展开方案。

反馈通知
弹窗 · Toast · 空状态

统一确认弹窗、危险操作二次确认、批量操作反馈的交互模式，避免多产品线出现风格迥异的"自制弹窗"。

导航结构
侧导航 · 顶导航 · 面包屑

针对美团云多层级产品结构，设计支持 2-4 级菜单的导航方案，并规范页面标题区、操作区的统一布局模式。

对现有产品页面进行系统性改版，在不改变功能的前提下，通过信息架构优化与视觉密度调整，让页面"更好用、更好看"。
✗ 改版前
信息堆砌，无视觉层级，关键数据难以扫视
操作按钮散落各处，主次不明，误操作频发
各产品线字号、颜色随意，视觉噪音极大
表格列宽不合理，关键信息被截断
空状态、加载态缺失，用户困惑
✓ 改版后
建立清晰信息层级，关键指标一眼可见
主操作突出，危险操作二次确认，误操作↓
统一视觉语言，信息密度提升 20%+
智能列宽分配，内容完整展示
完善各状态设计，体验闭环
典型改版场景
📊

监控大盘

重构指标卡片布局，引入趋势迷你图与状态色系，将原本密密麻麻的数字界面转变为可快速扫视的状态大盘。

📋

资源列表

统一资源列表的列定义优先级（名称→状态→关键参数→操作），批量操作区固定在顶部，大幅提升运维效率。

⚙️

配置中心

将嵌套层级过深的配置页拆分为分步式表单，每步聚焦单一任务，配置完成率显著提升。

建立设计师与研发的协同工作流，通过统一标注规范和组件文档，大幅降低研发还原成本，实现设计交付标准化。
📐

标注规范统一

制定间距、字号、颜色的标注规则，设计稿输出格式标准化，研发按规范还原，减少反复沟通。

📚

组件文档体系

每个组件配套使用说明、交互说明、状态说明，形成可检索的设计知识库，新成员上手成本降低 60%。

交付流程优化
Before

改版前 — 高沟通成本

设计师口头描述交互逻辑，研发自行理解，还原偏差大，需多轮 Review 返工，平均一个页面需要 3-5 次反复确认。

After

改版后 — 规范驱动交付

设计稿标注完整，组件文档自助查阅，研发直接对照组件库还原，Review 轮次从 3-5 次降至 1-2 次，交付效率提升 40%+。

推广与落地协同
